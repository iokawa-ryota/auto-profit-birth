## 🎯 プロジェクト目的

### 主目的

Binanceのグリッド取引機能等を用いて、市場が停滞している期間中に**2～3%の利益を機械的に短期回転**で積み上げ、軍資金を確保する。

### 副目的

長期的には、歴史的な大底（-60%下落）で大量購入できる資金を準備する。

サイクル的な大底での「本命買い」を確実に実行する

### 実行方法

| 項目                 | 内容                                                 |
| -------------------- | ---------------------------------------------------- |
| **プラットフォーム** | SBI VCトレード                                       |

2. **クジラ監視**: $55,000 付近の買い壁（Depth）確認後、分散指値
3. **タイミング**: 2026年10月（予想される底打ち時期）直前まで待機

### メリット

- ✓ SBIの信頼性による資産保護
- ✓ 入出金手数料無料で「絶対防衛資金」を効率的に管理
- ✓ 一度設定すれば、自動で大底を狙える

板の厚さを利用して、2～3%の利益をコツコツ積み上げ、軍資金を増やす

### 実行方法

| 項目                 | 内容                                                         |
| -------------------- | ------------------------------------------------------------ |
| **プラットフォーム** | Binance Japan                                                |
| **通貨ペア**         | BTC / USDT（テザー建て）                                     |
| **タイムフレーム**   | イントラデイ（数時間～1日）                                  |
| **エントリー戦略**   | 直近サポート（$62,000 や $68,000）の「買い壁」の少し上で拾う |
| **利確戦略**         | 購入価格から +2～3% で機械的に利確指値を設定                 |

### 待機資金の最大化

1. **Simple Earn（フレキシブル）**: 指値待機中の USDT は Simple Earn に預け、微益（金利）を得る
   - 利息発生 → 複利効果
   - 流動性確保 → 千早い約定に対応

2. **機動力確保**: 注文実行時に即座に Simple Earn から解除、現物ウォレットへ戻す

### 日次オペレーション

```

Midday (昼)
├─ 約定チェック
├─ 利確指値を即座に設定
└─ 次のエントリーポイント監視

Evening (夜)
├─ 日次利益計算
├─ Simple Earn への再投下判定
└─ 翌日シナリオプラン

Night (夜間)
├─ 米国市場の値動き監視
├─ 急騰・急落アラート設定

### 計算

```
1日の利益 = $10,000 × 2.2% × 8回 = $1,760
1ヶ月の利益 = $1,760 × 20営業日 = $35,200
年間利益 = $35,200 × 12ヶ月 = $422,400

利益率: 422,400 / 10,000 = 4,224% (!!)

## 🔧 トラブル対応フロー

### Q1: "Insufficient Balance" エラーが出た

**原因**: 口座にUSDTが不足している

**対応**:

1. Simple Earn から即座に USDT を引き出す
2. 現物ウォレットで確認
3. 必要に応じて、入金を実施

### Q2: 急騰・急落が発生した

**原因**: 市場イベント・ニュース

**対応**:

1. **クジラ監視**: $55,000 突破したか？
2. **支援** → **停止**: 上昇トレンドなら戦略停止、様子見
3. **下落継続** → **継続**: グリッド拡大・エントリー口数増

### Q3: 連日赤字が続いている

**原因**: 市場が大きく動いている、またはトレード数が少ない

**対応**:

1. **センチメント確認**: 強気度が低下していないか？
2. **グリッド調整**: サポート・レジスタンスを再計算
3. **休場判定**: 一旦、取引を停止し、次のセットアップを待機
